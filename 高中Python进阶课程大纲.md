# 高中 Python 进阶课程大纲

## 一、课程定位

本课程面向已经完成 Python 基础、函数、数据结构、面向对象、基础项目实战的高中生，目标不是单纯继续堆新知识点，而是进入“理论理解 + 项目实践 + 创新表达”并重的进阶阶段。

课程核心方向：

- 用算法和计算思维提升编程深度
- 用真实项目提升工程能力
- 用 AI 与创意课堂提升学习兴趣
- 用作品展示训练表达、复盘与创新能力

## 二、适合对象

- 已完成 Python 基础语法学习
- 能阅读和修改简单 Python 程序
- 学过函数、列表、字典、文件操作、异常处理、面向对象
- 做过至少 1 到 2 个基础项目，例如记账系统、贪吃蛇、五子棋等

## 三、课程总目标

完成本课程后，学生应能够：

1. 理解常见算法思想，并能将其用于项目问题求解。
2. 独立分析一个中小型项目的功能结构与数据结构。
3. 能基于已有项目进行二次开发和功能升级。
4. 初步掌握 Web 应用、数据分析和 AI 应用的基本实现方式。
5. 能以作品、文档和演示的形式表达自己的设计思路。

## 四、课程设计原则

### 1. 理论与实操并重

每个主题都包含：

- 理论课：理解概念、模型和方法
- 实操课：把理论变成代码或作品

### 2. 项目驱动

所有新知识都尽量通过项目场景引入，而不是孤立讲授。

### 3. 创新体验

课堂中加入：

- 可视化演示
- 对抗挑战
- 角色扮演
- 拆解现成程序
- AI 辅助实验
- 作品发布与路演

### 4. 适合高中阶段

避免过重的底层工程负担，不把重点放在复杂部署、分布式训练等高门槛方向，而强调“理解原理、做出作品、形成方法”。

## 五、课程模块总览

本课程建议按 16 周展开，共 8 个主题模块，每个主题 2 次课。

| 模块 | 主题 | 核心关键词 |
|------|------|------------|
| 模块 1 | 算法思维与可视化 | 搜索、排序、复杂度、BFS |
| 模块 2 | 游戏 AI 与策略设计 | 状态、规则、启发式、路径规划 |
| 模块 3 | 数据建模与信息系统 | 类设计、数据结构、持久化 |
| 模块 4 | Web 应用开发 | Flask、路由、模板、表单 |
| 模块 5 | 数据分析与可视化表达 | CSV、pandas、图表、结论 |
| 模块 6 | 自动化工具开发 | 文件处理、批处理、异常、效率 |
| 模块 7 | AI 应用开发入门 | Prompt、知识库、工作流、助手 |
| 模块 8 | 综合创新项目 | 选题、迭代、演示、复盘 |

## 六、16 周完整教学安排

---

## 模块 1：算法思维与可视化

### 第 1 周：算法是什么，为什么程序不只是“能跑就行”

**理论重点**

- 算法与程序的区别
- 输入、输出、规则、步骤
- 时间复杂度的直觉理解
- 为什么不同写法效率差异很大

**实操内容**

- 用 Python 实现线性查找与二分查找
- 比较不同查找方法的执行效果
- 制作简单的查找过程打印版

**创新体验**

- 课堂真人模拟“查找过程”
- 学生扮演数据，老师扮演算法控制器

**课后任务**

- 完成“猜数字中的二分思想”练习
- 写一段短文解释“为什么二分查找更快”

### 第 2 周：排序、搜索与算法可视化

**理论重点**

- 冒泡排序、选择排序的思路
- BFS 的基本思想
- 图和网格的初步理解

**实操内容**

- 编写排序可视化程序
- 用二维网格展示 BFS 找路过程
- 将排序或 BFS 结果输出为动画或分步打印

**创新体验**

- “算法解说员”活动：学生边运行边讲述算法过程

**课后任务**

- 选择一个算法，制作自己的讲解版代码

---

## 模块 2：游戏 AI 与策略设计

### 第 3 周：从游戏规则到“会思考的程序”

**理论重点**

- 游戏中的状态、动作、规则
- 决策程序与普通功能程序的区别
- 启发式策略的基本思想

**实操内容**

- 分析 `贪吃蛇` 项目的结构
- 找出蛇、食物、地图、分数、胜负之间的关系
- 为已有 AI 写出策略说明文档

**创新体验**

- “读代码侦探课”：先不给答案，让学生自己找 AI 决策逻辑

**课后任务**

- 画出贪吃蛇项目的模块关系图

### 第 4 周：升级游戏 AI

**理论重点**

- 贪心策略与安全策略的差异
- BFS 在游戏寻路中的应用
- 如何评价 AI 的“聪明程度”

**实操内容**

- 升级 `贪吃蛇` AI
- 增加速度变化、关卡或障碍
- 对比不同 AI 策略效果

**创新体验**

- 人机对战挑战赛
- 学生记录 AI 胜率并分析原因

**课后任务**

- 写一份“如何改进游戏 AI”的方案

---

## 模块 3：数据建模与信息系统

### 第 5 周：现实问题如何变成程序结构

**理论重点**

- 什么是数据建模
- 如何从现实对象抽象出类
- 属性、方法、状态、关系

**实操内容**

- 分析 `个人记账系统`
- 拆解 `Account`、`Transaction` 等类的职责
- 重新设计一个“学生成长记录系统”或“任务管理系统”的类结构

**创新体验**

- “先设计，后编码”训练
- 学生先画草图，再开始写程序

**课后任务**

- 完成一个小系统的类图或结构图

### 第 6 周：从基础系统到完整应用

**理论重点**

- 持久化的意义
- JSON / CSV 的适用场景
- 查询、统计、修改、删除的系统设计

**实操内容**

- 升级现有记账系统
- 增加修改记录、删除记录、日期筛选、分类统计
- 可选扩展：导出 CSV

**创新体验**

- “产品经理模式”：学生自己提出功能需求，再实现其中 2 到 3 个

**课后任务**

- 写一份系统更新日志

---

## 模块 4：Web 应用开发

### 第 7 周：网页背后的程序逻辑

**理论重点**

- Web 应用和本地脚本的区别
- 路由、请求、响应
- 模板、表单、前后端基本协作

**实操内容**

- 阅读并运行 Flask 项目
- 理解 `web_app.py`、模板、静态文件的关系
- 修改页面文案和表单结构

**创新体验**

- “拆网站课”：让学生观察一个网页，倒推它需要哪些程序结构

**课后任务**

- 设计一个自己的小网站主题

### 第 8 周：做一个可展示的 Web 项目

**理论重点**

- CRUD 的基本思想
- 用户交互体验基础
- 页面结构与信息组织

**实操内容**

- 完成一个 Flask 小项目
- 题目可选：
  - 学生作品展示站
  - 班级活动报名系统
  - 学习打卡系统
  - 简易留言板

**创新体验**

- “发布作品”课堂：每位学生展示自己的网站功能与页面设计

**课后任务**

- 补充项目 README 或使用说明

## 模块 5：数据分析与可视化表达

### 第 9 周：数据不只是数字，而是信息

**理论重点**

- 数据分析的基本流程
- 数据清洗、分类、统计
- 平均值、中位数、最大最小值、趋势

**实操内容**

- 读取 CSV 数据
- 统计成绩、消费、阅读、运动等数据
- 输出汇总结果

**创新体验**

- 选择学生自己熟悉的数据主题，提高代入感

**课后任务**

- 整理一份自己的小数据集

### 第 10 周：让数据“会说话”

**理论重点**

- 图表类型的选择
- 如何从图表得出结论
- 数据表达中的误导与规范

**实操内容**

- 使用 `pandas` 和 `matplotlib`
- 绘制柱状图、折线图、饼图
- 完成一页数据分析展示

**创新体验**

- “数据新闻”课堂：学生用图表讲一个结论

**课后任务**

- 完成一个迷你数据报告

## 模块 6：自动化工具开发

### 第 11 周：程序如何帮人节省时间

**理论重点**

- 自动化脚本的价值
- 文件读写、批处理、异常处理
- 程序鲁棒性的基本概念

**实操内容**

- 编写文件批量重命名工具
- 编写文件分类整理工具
- 增加异常处理和日志输出

**创新体验**

- 用真实文件夹做案例，让学生感受到自动化的实际价值

**课后任务**

- 设计一个“我身边可以自动化的问题”

### 第 12 周：做一个真实可用的小工具

**理论重点**

- 工具型项目的功能边界
- 输入校验
- 错误恢复与用户提示

**实操内容**

- 选做一个自动化工具：
  - 成绩表整理工具
  - 文本批量清洗工具
  - 图片/文件批量归档工具
  - 学习资料自动分类工具

**创新体验**

- “效率挑战”：对比人工操作和脚本处理的时间差

**课后任务**

- 优化工具并录制使用演示

## 模块 7：AI 应用开发入门

### 第 13 周：大模型到底能做什么，不能做什么

**理论重点**

- 大模型的基本能力边界
- Prompt 的作用
- 知识库问答与普通对话的区别
- AI 工具不是“魔法”，而是“能力 + 数据 +流程”

**实操内容**

- 设计不同 Prompt 并测试效果
- 对比模糊提问与精确提问的结果
- 体验 AI 在总结、分类、改写中的应用

**创新体验**

- “Prompt 对照实验”：同一任务用不同提示词测试结果

**课后任务**

- 设计 3 组高质量 Prompt

### 第 14 周：做一个自己的 AI 小助手

**理论重点**

- 知识库问答的基本思路
- 工作流与智能体的初步概念
- 如何让 AI 服务具体场景

**实操内容**

- 设计一个 AI 助手方案
- 题目可选：
  - 课程问答助手
  - 英语作文点评助手
  - 阅读摘要助手
  - 学习计划建议助手

**创新体验**

- “AI 产品经理”课堂：学生要解释这个助手解决了什么问题

**课后任务**

- 完成助手设计说明书

## 八、课堂创新设计建议

为了让课程体验更有新鲜感，建议固定加入以下机制：

### 1. 成品反推

先展示一个能运行的程序，再让学生推测：

- 这个程序需要哪些模块
- 数据怎么存
- 哪些地方会出错

### 2. 挑战任务卡

每节课布置一个“新增功能挑战”：

- 增加一个筛选条件
- 增加一个统计图
- 增加一个按钮
- 改进一个 AI 策略

### 3. 双角色协作

两人一组：

- 一人负责编码
- 一人负责测试、记录和讲解

### 4. AI 对照实验

同一个任务分别采用：

- 自己独立完成
- 借助 AI 辅助完成

对比：

- 速度
- 正确率
- 可理解性
- 可维护性

### 5. 阶段展示

每完成一个模块都做一次 mini demo，而不是只在期末展示。

## 九、评价方式建议

建议采用过程性评价，而不是只看最终代码。

### 评价维度

- 理论理解
- 编码实现
- 调试能力
- 结构设计
- 创新表达
- 项目展示

### 参考比例

- 平时练习与课堂表现：30%
- 模块项目：30%
- 阶段展示：20%
- 最终综合项目：20%

## 十、教师实施建议

### 1. 每个模块都要留“可见成果”

不要只讲概念，每个主题结束后都要有一个能运行、能展示、能解释的作品。

### 2. 控制难度递增

高一学生可以做出很好的作品，但不适合持续陷入过重的工程细节。要多给清晰边界和阶段目标。

### 3. 鼓励学生改旧项目

“重构旧项目”比“永远做新项目”更能体现真正的进步。

### 4. 理论一定要回到作品

每讲一个概念，都要回答这三个问题：

- 它解决什么问题
- 它在项目里出现在哪里
- 不用它会怎样

## 十一、课程最终成果建议

课程结束时，建议学生至少形成以下成果：

- 1 个算法可视化作品
- 1 个完整信息系统或 Web 小项目
- 1 个数据分析或自动化工具
- 1 个 AI 应用设计或原型
- 1 份个人项目作品集文档

## 十二、总结

这门后续课程的重点，不是让学生“学更多零散知识”，而是帮助他完成从“会写代码”到“会做项目、会解释设计、会形成作品”的过渡。

如果执行得当，这套课程不仅能继续提升 Python 能力，也能让学生逐步建立：

- 计算思维
- 工程意识
- 创新意识
- 表达与展示能力

这比单纯再多讲几个语法点更有长期价值。
